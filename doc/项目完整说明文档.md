## 项目概述

### 基本信息
- **项目名称**: 坦克大战 (Tank Battle Game)
- **灵感来源**: 经典NES游戏《坦克大战》(Battle City / Tank 1990)
- **开发语言**: Python 3.8+
- **游戏引擎**: Pygame 2.6.0+
- **项目类型**: 2D像素风格射击游戏 + AI智能系统

### 项目特色
这是一个现代化的坦克大战游戏实现,融合了**强化学习(Q-Learning)**和**遗传算法**的混合AI系统。游戏不仅还原了经典NES版本的像素风格和核心玩法,还加入了智能敌人学习系统、实时性能监控和全面的测试覆盖。

### 核心亮点
- 🎮 **经典还原**: 完美复刻NES坦克大战的像素风格和游戏机制
- 🤖 **智能AI**: Q-Learning + 遗传算法混合系统,敌人能自主学习战术
- 🎯 **AI玩家坦克**: 2个玩家坦克完全由AI控制,每个可承受10次射击
- 🧬 **渐进式进化**: 三阶段渐进式AI进化策略,避免早期数据不足问题
- 🔄 **自动重启**: 所有坦克击杀后自动重新开始,AI持续进化
- 📊 **实时监控**: FPS、内存、CPU、AI决策时间等性能指标实时追踪
- 🧪 **测试完备**: 全面的单元测试覆盖,确保代码质量
- 🏗️ **模块化架构**: 清晰的代码组织,易于维护和扩展
- 🔧 **全局AI实例**: 单一全局HybridAgent实例,所有敌方坦克共享,避免数据冲突
- 📝 **完善日志系统**: 日志配置提前到导入之前,确保所有模块日志正常输出

## 核心功能

### 1. 游戏系统

#### 地图系统
- **地图尺寸**: 20列 × 15行 (640×480像素)
- **瓦片大小**: 32×32像素
- **地形类型**:
  - `0` - 空地
  - `1` - 砖墙(可破坏)
  - `2` - 钢块(不可破坏)
  - `3` - 草地(隐蔽)
  - `4` - 基地(需要保护)

#### 坦克系统
- **玩家坦克**: 
  - 数量: 2个 (AI控制)
  - 初始位置: (7×32, 13×32) 和 (8×32, 13×32)
  - 生命值: 10点 (可承受10次射击)
  - 射击冷却: 350ms
  - 移动速度: 2像素/帧
  - 控制方式: AI自主决策(每300ms)
  
- **敌人坦克**:
  - 最大数量: 10个
  - 候选总数: 20个
  - 生命值: 1点
  - 射击冷却: 1200ms
  - 移动速度: 2像素/帧
  - AI决策周期: 300ms

#### 子弹系统
- **子弹速度**: 8像素/帧
- **子弹尺寸**: 6×6像素
- **碰撞类型**:
  - 击中砖墙 → 摧毁砖块 + 爆炸特效
  - 击中钢块 → 爆炸特效 (钢块无损)
  - 击中敌人 → 敌人爆炸 + 得分+100
  - 击中玩家坦克 → 玩家坦克HP-1 + 爆炸特效
  - 击中基地 → 游戏结束
  - 玩家坦克HP≤0 → 坦克被摧毁
  - 所有玩家坦克被摧毁 → 游戏结束
  - 所有坦克(玩家+敌人)被击杀 → 自动重新开始

### 2. AI智能系统

#### Q-Learning核心

**状态空间设计** (6维离散化,共432种状态):
```
状态向量 = (rel_x, rel_y, distance_cat, direction, has_obstacle, has_ally)

rel_x:          玩家在左(0) / 同列(1) / 玩家在右(2)
rel_y:          玩家在上(0) / 同行(1) / 玩家在下(2)
distance_cat:   近<100(0) / 中100-250(1) / 远>250(2)
direction:      上(0) / 右(1) / 下(2) / 左(3)
has_obstacle:   无(0) / 有(1)
has_ally:       无(0) / 有(1)
```

**动作空间**: 4个移动方向 (上/右/下/左)

**Q值更新公式**:
```
Q(s,a) ← Q(s,a) + α × [r + γ × max(Q(s',a')) - Q(s,a)]

其中:
  α (学习率) = 0.2 (可遗传优化)
  γ (折扣因子) = 0.95 (可遗传优化)
  r = 即时奖励
  s' = 下一个状态
  a' = 下一个动作
```

**经验回放机制**:
- **缓冲区容量**: 5000条经验
- **采样策略**: 优先级采样(TD误差驱动)
- **批处理大小**: 16条/次
- **优先级计算**: priority = |TD_error| + 0.01
- **重要性采样**: 权重修正避免偏差

#### 奖励系统设计

**事件奖励** (最高优先级):
| 事件 | 奖励值 | 触发条件 |
|------|--------|----------|
| 击杀玩家 | +15.0 | 敌人子弹击杀玩家 |
| 伤害玩家 | +3.0 | 敌人子弹击中玩家 |
| 敌人死亡 | -10.0 | 敌人被玩家击杀 |

**角色距离奖励**:
| 角色 | 条件 | 奖励值 | 战术意图 |
|------|------|--------|----------|
| 攻击者 | 距离<80 | +2.0 | 近身战斗 |
| 攻击者 | 距离80-150 | +1.0 | 接近中 |
| 攻击者 | 距离>300 | -0.5 | 太远惩罚 |
| 侧翼者 | 距离120-250且45°角 | +2.0 | 理想侧翼 |
| 侧翼者 | 距离>350 | -0.3 | 脱离战场 |
| 压制者 | 距离200-350 | +1.0 | 控制区域 |
| 压制者 | 距离<100 | -0.5 | 太近暴露 |

#### 动态角色分配

**三种战术角色**:

1. **攻击者 (Aggressor)**
   - 选择标准: 距离玩家最近的敌人
   - 战术目标: 直接追击,最短路径
   - 最优距离: <80像素

2. **侧翼者 (Flanker)**
   - 选择标准: 与玩家呈45度角的敌人
   - 战术目标: 侧翼包抄,计算垂直偏移点
   - 最优距离: 120-250像素
   - 包抄算法: perp_x = -dy/distance×100, perp_y = dx/distance×100

3. **压制者 (Suppressor)**
   - 选择标准: 控制地图中心位置的敌人
   - 战术目标: 封锁路径,控制要道
   - 最优距离: 200-350像素

#### 遗传算法优化

**优化参数**:
| 参数 | 范围 | 默认值 | 作用 |
|------|------|--------|------|
| 学习率α | 0.1~0.3 | 0.2 | Q值更新速度 |
| 折扣因子γ | 0.9~0.99 | 0.95 | 远期奖励权重 |
| 探索率ε | 0.1~0.4 | 0.3 | 探索vs利用 |

**适应度函数**:
```
Fitness = survival_time × 0.2
        + enemies_killed × 8
        - player_damage × 1.5
        + team_coordination × 3
```

## AI系统详解

### 工作流程

#### 1. 初始化阶段
```
游戏启动 → 配置日志系统 (logging.basicConfig)
    ↓  // ✅ 关键：日志配置必须在所有导入之前
初始化Pygame → 加载资源 → 创建游戏实例
    ↓
初始化AI系统 (QLearningAgent + GeneticOptimizer + PerformanceOptimizer)
    ↓
创建全局 HybridAgent 实例 (单例模式)
    ↓
重置游戏状态 → 进入游戏循环
```

**关键修复** (2026-04-10):
- ❌ 旧设计：`logging.basicConfig()` 在导入 `tank_ai` 之后调用 → tank_ai 中的日志不生效
- ✅ 新设计：`logging.basicConfig()` 提前到所有导入之前 → 所有模块日志正常输出

#### 2. 游戏循环 (每帧)
```
1. 事件处理 (键盘输入、窗口事件)
2. AI决策 (每300ms)
   - 记录上一帧经验
   - 调用AI进化 (evolve_before_new_game)  // ✅ 统一方法，替代 record_game_stats
   - 角色分配
   - Q-learning决策
   - 执行移动
   - 尝试射击
3. 物理更新 (子弹移动、碰撞检测、爆炸特效)
4. 渲染绘制 (地图、坦克、子弹、特效、UI)
5. 性能监控 (FPS、内存、缓存命中率)
6. 帧率控制 (clock.tick(60))
```

#### 3. 事件触发学习
- **敌人死亡时**: learn_from_death(history) → 从历史序列学习 → 经验回放
- **玩家被击杀时**: 所有存活敌人获得击杀奖励(+15)
- **玩家受伤时**: 最近敌人获得伤害奖励(+3)

#### 4. 渐进式AI进化策略 (三阶段)

```
┌─────────────────────────────────────────────────────┐
│ 阶段1: 数据累积 (1-3局)                              │
│ 📊 策略: 不进化，只收集数据                            │
│ 原因: 避免基于少量数据做出错误决策                     │
│ 探索率: 每局衰减5%                                    │
├─────────────────────────────────────────────────────┤
│ 阶段2: 轻量进化 (4-9局)                              │
│ 🌱 策略: 基于现有种群低强度变异                        │
│ 方法: 保留精英50%，变异幅度30%                         │
│ 经验回放: 2次                                         │
├─────────────────────────────────────────────────────┤
│ 阶段3: 标准进化 (10局+)                              │
│ 🧬 策略: 完整遗传算法                                 │
│ 数据: 最新10局真实数据                                 │
│ 经验回放: 3次                                         │
└─────────────────────────────────────────────────────┘
```

**进化触发时机**:
- 所有坦克击杀后自动触发
- 游戏结束界面手动重启时触发
- 老鹰被摧毁时立即触发

**进化方法**: `evolve_before_new_game(game_stats)`
- ✅ 统一入口：所有游戏结束场景都调用此方法
- ✅ 自动衰减：探索率每局递减5%
- ✅ 渐进进化：根据局数自动选择进化强度
- ✅ 自动保存：进化完成后自动保存 checkpoint

**进化流程**:
```
游戏结束 → 收集统计数据
    ↓
调用 evolve_before_new_game(game_stats)
    ↓
探索率衰减5% (exploration_rate *= 0.95)
    ↓
判断阶段:
  ├─ 1-3局: 累积数据，不进化
  ├─ 4-9局: 轻量进化 (保留精英，低强度变异)
  └─ 10局+: 标准进化 (完整遗传算法)
    ↓
更新Q-learning参数 → 经验回放 → 自动保存 → 日志输出
```

**优势**:
- ✅ 避免早期基于虚假数据的错误决策
- ✅ 逐步增加进化强度，平滑过渡
- ✅ 只使用真实数据，不生成虚拟样本
- ✅ 参数变化更稳定，AI行为一致
- ✅ 断点续训：自动加载/保存模型，支持跨程序运行持续学习

#### 5. 全局AI实例设计

**设计理念**: 所有敌方坦克共享同一个 `HybridAgent` 实例

**实现方式**:
```python
# main.py - reset_game()
if global_hybrid_agents is None:
    # 首次创建时加载 checkpoint
    global_hybrid_agents = [HybridAgent()]
else:
    # 重置游戏时，不清空缓冲区
    # 让数据跨局累积，由 evolve_before_new_game() 在进化时自动清空
    pass

# 所有敌方坦克分配同一个 AI 实例
for enemy in enemies:
    enemy_ais[enemy] = global_hybrid_agents[0]
```

**优势**:
- ✅ 避免重复计数：`games_played` 只在一个地方增加
- ✅ 数据一致性：所有坦克共享同一个 Q-table 和学习历史
- ✅ 内存优化：只维护一个 AI 实例，而非多个
- ✅ 持续学习：缓冲区跨局累积，进化后自动清空
- ✅ 断点续训：checkpoint 保存/加载，支持程序重启后继续学习

**关键修复** (2026-04-10):
- ❌ 旧设计：每个坦克独立 HybridAgent 实例 → 数据冲突、计数翻倍
- ✅ 新设计：单一全局 HybridAgent 实例 → 数据一致、计数准确

## 游戏机制

### 1. 地图设计
- **地图尺寸**: 20列 × 15行
- **地形类型**: 空地(0)、砖墙(1)、钢块(2)、草地(3)、基地(4)

### 2. 碰撞系统
- 坦克 ↔ 墙壁/边界: 阻止移动
- 玩家子弹 ↔ 砖墙: 摧毁砖墙 + 得分+30
- 玩家子弹 ↔ 敌人: 敌人爆炸 + 得分+100
- 敌人子弹 ↔ 玩家: 生命-1
- 任意子弹 ↔ 基地: 游戏结束

### 3. 得分系统
| 事件 | 得分 |
|------|------|
| 摧毁砖墙 | +30 |
| 击杀敌人 | +100 |
| 保护基地 | +500 |

### 4. 生命系统

#### 玩家坦克生命系统
- **初始HP**: 每个玩家坦克10点 (START_LIVES=10)
- **总HP**: 2个坦克 = 20点
- **受伤惩罚**: 每次被击中HP-1
- **坦克摧毁**: HP≤0时坦克被移除
- **游戏结束**: 所有玩家坦克被摧毁

#### 敌人坦克生命系统
- **初始HP**: 每个敌人坦克1点
- **被击中**: 立即爆炸并摧毁
- **重生机制**: 候选坦克>0时自动生成新敌人 (最多20个)

### 5. 自动重启机制

**触发条件**:
- 所有候选坦克被击杀 (`candidate_tanks ≤ 0`)
- 场上没有敌人 (`len(enemies) == 0`)

**重启流程**:
```
1. 记录当前游戏统计 (survival_time, enemies_killed, etc.)
2. 调用 AI 进化: agent.evolve_before_new_game(game_stats)  // ✅ 统一方法
3. 重置 AI 决策器 (ai_decider.reset())
4. 手动触发 GC (gc.collect())
5. 重置游戏到初始状态 (reset_game(global_hybrid_agents))  // ✅ 保留全局实例
6. 重置统计计数器
7. 增加游戏局数 (total_games += 1)
8. 继续游戏 (或达到最大局数50退出)
```

**关键修复** (2026-04-10):
- ❌ 旧设计：使用 `record_game_stats()` → 方法重复、探索率不衰减
- ✅ 新设计：统一使用 `evolve_before_new_game()` → 功能完整、探索率自动衰减
- ✅ 全局实例：`reset_game(global_hybrid_agents)` 保留 AI 实例，避免重新创建

**最大局数**: 50局 (可配置 `max_games`)

## 常见问题

### Q1: 游戏无法启动
1. 检查Python版本 (需要3.8+)
2. 检查依赖安装 (pygame, psutil)
3. 检查资源文件是否存在
4. 查看game.log日志

### Q2: 游戏运行卡顿
1. 增加CACHE_TIMEOUT到10
2. 减少MAX_ENEMIES到5
3. 关闭其他程序释放资源

### Q3: AI不学习
1. 检查exploration_rate > 0.05
2. 检查replay_buffer > 100
3. 增加训练局数到100
4. 查看控制台进化日志

### Q4: 内存占用过高
1. 减少REPLAY_BUFFER_SIZE到3000
2. 定期清理缓存
3. 重启游戏(F2)

### Q5: 玩家坦克为什么不响应键盘?
**这是正常行为!** 玩家坦克现在完全由AI控制。
- AI自动寻找最近敌人
- AI自动移动和射击
- 每300ms决策一次

### Q6: 游戏如何自动重启?
当**所有坦克**(2个玩家+20个敌人)都被击杀后:
1. 自动记录游戏统计
2. 自动进化AI模型
3. 自动重置游戏
4. 继续下一局
5. 最多运行50局
